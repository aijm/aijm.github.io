RISC-V

question

  1. 为什么会提出栈、堆这种东西，有什么作用？

    栈是为了保存函数调用时当前的寄存器状态，因为执行函数指令时可能会改变寄存器的值。堆是为了存储程序中创建的数据。

  2. I-format 怎么处理immediates > 12bits 的情况？

    用LUI等指令，先存入寄存器中

  3. B-format 怎么处理分支Label, 怎么存储label?怎么处理offset超出表示范围的情况?

    offset 超出范围，可以通过多个中间label 跳转

  4. U-format 有哪些指令？AUIPC 用于什么？

    U-format 有 LUI, AUIPC等指令，LUI表示Load Upper Immediate, LUI rd, ,与 addi rd, 结合可以将32bit 立即数放入寄存器中。AUIPC rd, 可以将PC与 相加，用于处理PC-relative addressing

  5. j, jr, jal, jalr 之间的关系？
    • jal rd, label : PC跳转到label处，且将PC+4存入rd寄存器
    • JALR rd, rs, immediate: PC跳转到 rs+immediate处,且将PC+4存入rd
    • j label 就是 jal x0, label;jr ra 就是 jalr x0, ra, 0

函数调用步骤

  1. 参数存入a0,a1,…
  2. 调用时需要保存return address, 需要保存的寄存器的值；存储这些值是因为调用的函数可能会改变这些值。
    • Return “instruction” address
    • Parameters
    • Space for other local variables
  3. 上面的值通过栈来保存，就是所谓的调用栈
  4. 通过 jal ra, func 调用函数，这条语句会把ra的值设为下一条指令的地址，同时把pc的值变为函数func的地址
  5. 执行函数，通过a0返回值，在最后 jr ra, 会将pc的值变为ra,返回到之前调用处的下一条指令
  6. 函数将返回值存入寄存器a0中，同时从栈中恢复原来保存的值，通过sp增加回收栈空间

instruction representation

计算机是不能直接执行汇编代码的，计算机能处理的只有机器码0、1,而汇编代码实际上就是为了方便编程，让程序员不用写复杂的0,1。所以每条汇编代码实际上都对应着相应的机器指令。
首先总结一下一个程序需要的所有指令的类型。

  • Arithmetic: add, addi, sub
  • Logical: sll, srl, sla, slli, srli, slai, and, or, xor, andi, ori, xori
  • Desicion branches: beq, bne, blt, bge
  • jump: j(jump label), jr(jump register)
  • Function call: jal ra, funLabel; return with jr ra
  • load, save with memory: lw, sw, lb, sb

对于32位机器，一条指令为32位，称为一个word, 计算机要根据不同的指令做出对应的操作，所以需要事先规定指令的格式。
对于RISC-V, 主要有以下几种instruction fromats:
由于共32个寄存器，所以寄存器可以用5位来表示。

  1. R-format for register-register arithmetic operations

add x8, x9, x10


  1. I-format for register-immediate arithmetic operations and loads

rs2 和 funct7 组成12bit $[-2^{11}, 2^{11}-1]$ 用于表示immediate 的值

addi x8, x9, 11


  1. S-format for stores
    sw rs2, immediate(rs1),由于store不需要写入寄存器，所以没有指令中rd部分不是寄存器，可以用于表示immediate

  2. B-format for branches (minor variant of S-format, called SB before)

需要比较rs1,rs2两个寄存器的值，并存储分支的label地址。关键是怎么编码label？可能一个label的地址距离非常远，12bit根本存不下。

可以通过PC-Relative Addressing 来实现，通过存储label距离PC的偏移量，为了兼容16bit指令，可以通过PC = PC + offset 得到label的地址。于是只需存储offset(byte)的值即可。为了兼容16bit指令，offset 是2的倍数，如offset = …, -4, -2, 0, 2, 4, …

由于是2的倍数，offset的最低位是0，所以实际上可以用12bit表示13bit,offset的表示范围为[-4096, 4094]。

beq x1, x2, label


  1. U-format for 20-bit upper immediate instructions
    为了解决之前I-format 中immediate只有12bit的问题。通过U-format先将高20位存入寄存器，实现32bit immediate的操作。相应汇编为:LUI, AUIPC, Li



  2. J-format for jumps (minor variant of U-format, called UJ before)

jal rd, label 需要存储下一条指令的地址到rd中，并且记录label与PC的offset

  1. I-format for jumps

调用函数不只是根据label去跳转，有时需要给出直接的地址。

JALR rd, rs, immediate 就是跳转到 rs+immediate的位置。
